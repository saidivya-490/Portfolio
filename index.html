  • Triage: Identified a potential shared dependency using Dynatrace Smartscape instead of treating them as isolated failures.

  • Prevention: Redesigned the archival process to run in smaller batches and implemented database resource governance to avoid future production impact.

Microservices Reporting Optimization Using SQL Materialized View

  • Triage: Analyzed reporting latency and collaborated with stakeholders to define performance targets for near real-time insights.

  • Fix: better alerting, reduced noise, clearer escalation path.Designed a reporting microservice backed by a SQL Server materialized view (indexed view) to pre-aggregate critical data. Updated .NET Core APIs to fetch from the optimized view instead of running runtime aggregations.
  • Prevention: Established a refresh strategy and monitoring for the materialized view and defined architectural guidelines to use pre-aggregated data for read-heavy workloads.

Database: Deadlocks / blocking causing API timeouts (SQL Server / Oracle)

  • Signal: Sudden increase in API latency + timeout errors; elevated DB CPU/locks during peak hours.
  • Triage: Reduced blast radius (temporary throttling / feature flag / queueing) and stabilized user-facing flows.
  • Diagnosis: Identified blocking chains and deadlock patterns using DB monitoring (locks, wait stats), correlated with Splunk traces to isolate the exact API endpoints and queries involved.
  • Decision: Prioritized a safe fix path: reduce lock duration first (query rewrite + indexing), then adjust transaction scope and isolation where needed.
  • Fix: Refactored the stored procedure/query to avoid wide scans, added targeted indexes, reduced transaction scope, and introduced idempotent retry for safe transient failures.
  • Prevention: Added DB guardrails (timeouts, alert thresholds), created a runbook for lock escalation, and added dashboards tracking waits/latency/error rates.
